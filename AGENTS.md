# AGENTS.md · Colvir · занятие 7

Учебный репозиторий для практики Git и GitHub.

## Отчёт

- Презентация живёт в одном файле: `index.html`
- Факты для наполнения слайдов — в `brief.md`
- В отчёте **24 пустых слайда** (`#slide-01` … `#slide-24`) — по одному на человека в команде
- Внизу страницы — справочник **шаблонов** (`#templates`), их не перезаписывай

## Стили

- Tailwind только через CDN: `https://cdn.tailwindcss.com`
- Только utility-классы Tailwind
- Без `<style>`, без `.css` файлов, без inline `style="..."`

## Промпт для старта

Когда участник готов работать, скажи:

> Выбери слайд — в отчёте 24 пустых места (слайды 1–24). Могу показать свободные и предложить подходящий шаблон.

Дальше:

1. Спроси номер слайда или предложи свободный (где ещё «Пустой слайд»)
2. Предложи один шаблон из списка ниже
3. Замени содержимое **только** выбранного `#slide-NN` — убери пунктирную рамку, вставь вёрстку по шаблону
4. Факты бери из `brief.md`, не выдумывай сверх брифа
5. После правки предложи коммит на русском, например: `Заполнил слайд 7 — три цифры`

## Шаблоны слайдов

Образцы вёрстки — в `index.html` в блоке `#templates`. Код каждого шаблона:

| ID | Название | Когда использовать |
|---|---|---|
| `template-cover` | Обложка | Титул, название отчёта, подзаголовок по центру |
| `template-metrics` | Три цифры | Три KPI в ряд: число + подпись |
| `template-columns` | Две колонки | Плюсы слева, минусы справа (или любые два блока) |
| `template-cards` | Три карточки | Три темы одного уровня: склад, реклама, поддержка |
| `template-list` | Список | Перечень пунктов с заголовком |
| `template-statement` | Один вывод | Вопрос руководству или главная мысль по центру |

Каждый шаблон — `<section class="min-h-screen ...">` на весь экран. При переносе в рабочий слайд сохраняй `id="slide-NN"` и `data-slide="N"`, меняй подпись «свободен» на имя участника или тему.
